# モバイルアプリ認証フロー：端末自動ID割り当て（匿名認証）

ユーザーが初回起動時にID/パスワードを入力することなく、アプリが自動的に端末を識別してアカウントを割り当てるフローです。主にカジュアルゲームや、利用開始のハードルを下げたいアプリで採用されます。

---

## 1. 概要
このフローは「匿名ログイン（Anonymous Login）」とも呼ばれます。ユーザーの手間を最小限に抑え、即座にサービスを開始できるのが特徴です。

## 2. 認証シーケンス

1.  **アプリ起動**: アプリがローカルストレージ（KeyChainやSharedPreferences）に既存のトークンがあるか確認。
2.  **ID生成/取得**:
    *   トークンがない場合、アプリ側で UUID を生成、またはOS提供の識別子（Vendor ID等）を取得。
3.  **サーバーへの登録**:
    *   アプリが識別子をバックエンドAPIに送信。
    *   サーバーは新しいユーザーレコードを作成し、そのユーザー専用の「アクセストークン（JWT等）」を発行。
4.  **トークンの保存**:
    *   アプリは受け取ったトークンをセキュアなストレージに保存。
5.  **リクエスト実行**:
    *   以降のAPIアクセスでは、このトークンを `Authorization` ヘッダーに付与。

## 3. メリットとデメリット

### メリット
*   **離脱率の低下**: 登録の手間がないため、ユーザーがすぐにアプリを利用できる。
*   **摩擦のない体験**: ユーザーに「アカウント作成」を意識させない。

### デメリット・リスク
*   **データ移行の困難さ**: 機種変更時にデータを引き継ぐために、後述の「アカウント連携」が別途必要になる。
*   **アプリ削除時のリスク**: トークンを保存している領域が削除されると、ユーザーが自分のアカウントに戻れなくなる可能性がある。
*   **セキュリティ**: 端末の識別子のみに依存する場合、なりすましのリスクを考慮する必要がある。

## 4. 推奨される補完策
*   **アカウントのアップグレード**: ユーザーが特定のレベルに達した際や、課金を行うタイミングで「メールアドレス設定」や「SNS連携」を促し、匿名アカウントから永続的なアカウントへ昇格させる導線を用意する。
